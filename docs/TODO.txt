## AUTHENTICATION & ACCESS (spec §2.1)
- [x] [SIMULATED] CPS authentication → Use standard form login instead
- [x] Fallback authentication (standard username/password)
- [x] Activity logging: track user identity, login time, logout time
- [x] User profiles with different access levels (full access vs read-only)
- [x] Patient access control (Art. L1110-4, L1110-12 CSP):
    - Every doctor (CHU or external) can only access their own assigned patients
    - Technical admins (ROLE_TECH_ADMIN): NO patient data access
    - PatientAccessVoter (VIEW_PATIENT) enforces per-patient access
    - isChuPractitioner deprecated — no longer used for access control
- [x] ROLE_MEDICAL_ADMIN removed — replaced by break-the-glass (bris de glace):
    - No role has blanket access to all patients (compliance with RGPD/CNIL)
    - Doctors can use break-the-glass for emergency access (30 min, justified, audited)
    - Nurses can also use break-the-glass with same constraints
    - Technical admins cannot use break-the-glass (no patient data access)
    - BreakTheGlassAccess entity logs: user, patient, justification, timestamps
    - AccessDeniedHandler redirects medical personnel to BTG form
    - Audit log records all BTG events
- [x] Split admin role: ROLE_TECH_ADMIN (system) — no medical admin role exists
- [x] ROLE_SUPER_ADMIN: inherits ROLE_TECH_ADMIN, full user/role management, can assign
      any role including ROLE_DOCTOR, ROLE_NURSE, ROLE_TECH_ADMIN, ROLE_TRANSPLANT_COORDINATOR
- [x] ROLE_TRANSPLANT_COORDINATOR: inherits ROLE_USER, can create/edit/view donors,
      link donors to patients, cannot create/edit/view patient data
- [ ] Find a way for doctors and nurses to be given access to patients (and maybe donors) other than being the creator of the patient and the donor
    - An idea is that a doctor can give and remove access to other doctors and nurses and those can't give access
    - A doctor can change his access to give it to another doctor (not nurse) to give their patient, doing so will remove the access to the patient from the first doctor and the second doctor will have the right to give access to other doctors
    - A super_admin (or normal admin, should check about it) should be able to give, move to someone else and remove full access and secondary access to users
